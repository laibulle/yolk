# The bridge

The bridge is the layer between the JavaScript runtime and native code. Yolk's bridge is a **buffer-only**, high-performance interface designed for zero-copy data transfer and end-to-end type safety.

## Zero-Copy Architecture

Unlike traditional bridges that rely on JSON serialization, Yolk uses raw memory buffers (**ArrayBuffer** in JS, **Data** in Swift) for all communication. 

### Why buffers?
1.  **Performance**: Eliminates the `JSON.stringify` and `JSON.parse` overhead, which is the primary bottleneck in cross-platform frameworks.
2.  **Memory**: Allows direct, shared memory access for large datasets (images, audio, video frames).
3.  **Predictability**: Byte-level control over data layout ensures consistent behavior across different JS engines (JSC vs. QuickJS).

## How a call works

When TypeScript calls a native module method, the arguments are packed into a binary buffer using the **YolkBin** protocol:

```
TypeScript                    Bridge                     Swift
──────────                    ──────                     ─────
playground.increment(1)
  │
  ├─▶ YolkBin.encode([1]) ──▶ ArrayBuffer
  │
  └─▶ __yolk_native_Playground("increment", buffer)
           │
           ├─▶ Create JS Promise (id=42)
           ├─▶ Return promise to caller  ──▶ TypeScript awaits
           │
           └─▶ Task {
                 // args is raw Data
                 await module.handle("increment", args)
                     │
                     ├─▶ YolkBin.decode(args) ──▶ [1.0]
                     │
                     └─▶ actor PlaygroundModule
                               │
                               └─▶ increment(step: 1.0) ──▶ PlaygroundState(...)
                 }
                     │
                     ├─▶ YolkBin.encode(state) ──▶ Data
                     │
                     └─▶ jsQueue.async {
                               // result is JS ArrayBuffer
                               __yolk.resolve(42, result)
                               drainMicrotasks()       ──▶ Promise resolves
                         }
```

## The YolkBin Protocol

Yolk implements a custom **Type-Length-Value (TLV)** protocol for binary serialization. It handles the following types natively at the byte level:

- **0x00**: Null / Undefined
- **0x01**: Boolean (1 byte)
- **0x03**: Number (Float64, 8 bytes)
- **0x04**: String (UTF-8 encoded)
- **0x05**: Array
- **0x06**: Object (Key-Value pairs)
- **0x07**: Raw Buffer (Zero-copy pass-through)

## Single Source of Truth (SSOT)

Yolk follows a reactive state model. The application state lives in JavaScript. Whenever the state mutates, the JS logic pushes the new state as a binary buffer to a native **Observer**:

```typescript
// JS notifies native of state change
const buffer = YolkBin.encode([state]);
globalThis.__yolk_native_Observer("onStateChanged", buffer);
```

On the native side, the generated **PlaygroundState** struct is automatically updated, triggering a reactive UI refresh in SwiftUI or Jetpack Compose.
